# Платформа для организации каршеринга

## Автор

Леонова Элина, ИКБО-10-23

## Описание проекта

Платформа для организации каршеринга — это SPA-приложение для аренды автомобилей с разделением ролей на пользователя и администратора. Пользователь работает с картой, бронированием, поездками и кошельком, а администратор дополнительно управляет автопарком, заявками, тарифами и бонусными зонами возврата.

Приложение состоит из:

- `frontend` — клиентская часть на React и TypeScript;
- `backend` — серверная часть на Django и Django REST Framework;
- `postgres` — база данных PostgreSQL;
- `nginx` — проксирование запросов.

## Возможности пользователя

- регистрация после подтверждения аккаунта администратором и авторизация;
- просмотр всех автомобилей на карте Москвы;
- бронирование автомобиля;
- совершение поездки с построенным маршрутом до точки назначения;
- расчет стоимости поездки и автоматическое списание итоговой суммы со счета;
- применение скидки при завершении поездки в бонусной зоне;
- просмотр активной брони, активной поездки и истории поездок;
- пополнение кошелька и просмотр истории операций.

## Возможности администратора

Администратор сохраняет весь пользовательский функционал и дополнительно получает отдельные разделы для управления системой.

- просмотр списка пользователей;
- подтверждение или отклонение заявок пользователей;
- просмотр списка, добавление, редактирование и удаление автомобилей;
- просмотр всех бронирований и поездок;
- настройка базового тарифа;
- настройка временных коэффициентов спроса;
- создание и редактирование бонусных зон возврата на карте.


## Используемые технологии

- React — библиотека для создания пользовательских интерфейсов.
- TypeScript — язык программирования, расширяющий JavaScript системой типов.
- Webpack — инструмент сборки frontend-приложения.
- HTML5, CSS3 — технологии для создания адаптивного пользовательского интерфейса.
- Django — веб-фреймворк для серверной части приложения.
- Django REST Framework — набор инструментов для создания REST API.
- PostgreSQL — система управления реляционной базой данных.
- Docker, Docker Compose — инструменты для контейнеризации и воспроизводимого запуска проекта.
- Nginx — веб-сервер и reverse proxy для frontend и backend на одном адресе.
- Yandex Maps JavaScript API — API для отображения карты, машин, маршрутов и пользовательских точек.

## Запуск проекта

1. Клонируйте репозиторий:

```powershell
git clone https://github.com/elinaleonovva/carsharing_app
cd carsharing_app
```

2. Создайте файл `.env` на основе шаблона:

```powershell
Copy-Item .env.example .env
```

3. Заполните в `.env` ключ:

```env
APP_YANDEX_MAPS_API_KEY=ваш_ключ_яндекс_карт
```
Без этого ключа карта не будет отображаться.

4. Запустите контейнеры:

```powershell
docker compose up --build
```

5. После запуска приложение будет доступно по адресу:

[http://localhost:8080](http://localhost:8080)

## CI/CD деплой на сервер

При пуше в ветку `master` GitHub Actions:

- собирает Docker-образы `elinaleonova/carsharing-backend`, `elinaleonova/carsharing-frontend` и `elinaleonova/carsharing-nginx`;
- отправляет их в Docker Hub;
- загружает на сервер только файл `docker-compose.yml`;
- выполняет на сервере `docker compose pull` и `docker compose up -d`.

### Что добавлено в репозиторий

- workflow: `.github/workflows/deploy.yml`;
- production compose: `docker-compose.server.yml`;
- Dockerfile для production nginx: `infra/nginx/Dockerfile`.

### GitHub Secrets

В настройках репозитория нужно создать:

- `DOCKERHUB_USERNAME` — логин Docker Hub;
- `DOCKERHUB_TOKEN` — access token Docker Hub;
- `SERVER_HOST` — IP или домен сервера;
- `SERVER_PORT` — SSH-порт сервера, обычно `22`;
- `SERVER_USER` — пользователь для SSH;
- `SERVER_SSH_KEY` — приватный SSH-ключ для входа на сервер;
- `SERVER_DEPLOY_PATH` — директория на сервере, где будет лежать `docker-compose.yml`, например `/opt/carsharing`;
- `POSTGRES_DB` — имя базы данных;
- `POSTGRES_USER` — пользователь PostgreSQL;
- `POSTGRES_PASSWORD` — пароль PostgreSQL;
- `DJANGO_SECRET_KEY` — секретный ключ Django;
- `DJANGO_ALLOWED_HOSTS` — список хостов через запятую, например `example.com,www.example.com,SERVER_IP`;
- `DJANGO_CSRF_TRUSTED_ORIGINS` — список origin через запятую, например `https://example.com,http://SERVER_IP`;
- `FRONTEND_URL` — публичный адрес frontend, например `https://example.com/`;
- `APP_YANDEX_MAPS_API_KEY` — ключ Yandex Maps API;
- `APP_YANDEX_MAPS_LANG` — язык карт, например `ru_RU`;
- `NGINX_PORT` — порт, который будет открыт на сервере, например `80`.

### Что должно быть на сервере

На сервере должны быть установлены:

- Docker;
- Docker Compose plugin (`docker compose`);
- открытый порт приложения, который указан в `NGINX_PORT`.

Для первого запуска достаточно создать директорию деплоя, например:

```bash
sudo mkdir -p /opt/carsharing
sudo chown $USER:$USER /opt/carsharing
```

После этого каждый `push` в `master` будет автоматически обновлять приложение на сервере.
